A Solução

Arquitetura Determinística de Baixa Alucinação

NavegAI usa uma arquitetura híbrida: IA generativa para entender intenção, mas Python puro para gerar SQL — minimizando alucinações.

🛡️ Baixa Alucinação

O LLM nunca gera SQL livre. Apenas extrai intenção em JSON estruturado. O SQL é montado por Python com validação rigorosa de views e colunas.

📡 12 Views Inteligentes

Mapeamento completo de vw_projetos_inteligencia, vw_ia_engajamento_acesso, vw_ia_indicadores_pem e outras 9 views.

⚡ Resposta em 3 Segundos

Pipeline otimizado: extração de intenção → construção SQL → execução → visualização. Tudo em menos de 3 segundos.

🔐 Segurança por Design

Operadores SQL validados, views protegidas, sanitização de inputs. Nenhum dado do usuário é concatenado diretamente no SQL.

Kimi Claw

Debug & Refatoração Inteligente

Agente especializado em análise de código. No NavegAI, foi usado para debugar queries SQL, corrigir gráficos ECharts e refatorar componentes críticos.

  • Correção de SQL com CASE WHEN para faixas temporais
  • Refatoração do motor de visualização ECharts
  • Otimização do prompt system do extrator de intenção
  • Validação de segurança nas queries geradas

Agent Swarm

Execução Paralela de Tarefas

Múltiplos agentes trabalhando simultaneamente: um otimiza o backend SQL, outro ajusta o frontend Streamlit, outro revisa UX e documentação.

  • Backend: Otimização do ConstrutorSQL e MotorCalculo
  • Frontend: Ajustes de responsividade e tema dark
  • UX: Melhoria na experiência de chat e feedback visual
  • Docs: Geração automática de README e storytelling
